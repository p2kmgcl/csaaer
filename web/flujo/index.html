csaaer - Cifrado de flujo

Demostraciones y estudios sobre algoritmos criptográficos

Introducción

Los cifrado de flujo son algoritmos criptográficos modernos, los cuales cifran un mensaje original combinándolo, usando la operación XOR, con un flujo de datos pseudoaleatorio del mismo o incluso mayor tamaño, generando así un texto cirado con clave secreta.

La secuencia de datos cifrante (el flujo de datos) se obtiene de una clave secreta K que solo conocen el emisor y el receptor. Para conseguir que este flujo sea de gran tamaño, se utiliza una semilla de valor N que generará flujos de valor superior a 2^N, consiguiendo así reducir la clave que los usuarios deberán conocer.

Algo de historia

El primer algoritmo de flujo fue el Cifrado Verman, creado en 1917 en los laboratorios AT&T Bells Labs por Gilbert Verman.

Joseph Mauborgne contribuyó tras la creación del cifrado proponiendo el uso de una cinta de papel donde se almacenase información aleatoria. Este añadido al anterior cifrado propició la creación de la libreta de un solo uso. Este método de cifrado es irrompible, tal y como demostró Claude Shannon, pero también requiere libretas perfectamente aleatorias y tan largas como el propio mensaje, lo que dificulta su implementación y suele concluir en errores que posibilitan un ataque criptográfico.

Uno de los últimos y más conocidos cifrados de flujo basados en el Verman es el RC4, que se utiliza en muchos protocolos importantes en Internet.
